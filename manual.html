1. Общее описание проекта

1.1 Цели и задачи

Матричный калькулятор — это интерактивное веб-приложение, предназначенное для выполнения основных операций линейной алгебры. Программа позволяет студентам, инженерам и всем интересующимся математикой выполнять вычисления с матрицами, решать системы линейных алгебраических уравнений (СЛАУ) и производить операции над векторами — всё в рамках единого удобного интерфейса.

Основные цели проекта:

  • Предоставить инструмент для быстрого и точного выполнения матричных операций
  • Реализовать три метода решения СЛАУ (Гаусса, Крамера, матричный) с пошаговым отображением
  • Поддержать операции над векторами, включая скалярное, векторное и смешанное произведения
  • Обеспечить детальный вывод промежуточных шагов решения для учебных целей
  • Создать современный, адаптивный интерфейс, работающий в любом браузере без установки

2. Математические основы: операции над матрицами

Матрицей размера m × n называется прямоугольная таблица чисел, содержащая m строк и n столбцов. Элемент матрицы, стоящий на пересечении i-й строки и j-го столбца, обозначается aij.

2.1 Сложение и вычитание матриц

Сложение и вычитание определено только для матриц одинакового размера. Результатом является матрица того же размера, каждый элемент которой равен сумме (разности) соответствующих элементов исходных матриц.

Cij = Aij ± Bij
где i = 1, 2, …, m; j = 1, 2, …, n; матрицы A и B имеют размер m × n

Условия применимости:

  • Матрицы A и B должны иметь одинаковое количество строк
  • Матрицы A и B должны иметь одинаковое количество столбцов
  • Результирующая матрица C имеет тот же размер m × n

Свойства операции сложения:

  • Коммутативность: A + B = B + A
  • Ассоциативность: (A + B) + C = A + (B + C)
  • Существование нулевого элемента: A + O = A, где O — нулевая матрица
  • Существование противоположного элемента: A + (−A) = O

2.2 Умножение матрицы на число

При умножении матрицы на число (скаляр) k, каждый элемент матрицы умножается на это число. Операция определена для матрицы любого размера и любого вещественного числа.

Cij = k · Aij
где k — вещественное число, i = 1, 2, …, m; j = 1, 2, …, n

Свойства:

  • 1 · A = A (умножение на единицу не меняет матрицу)
  • 0 · A = O (умножение на ноль даёт нулевую матрицу)
  • (k₁ · k₂) · A = k₁ · (k₂ · A) (ассоциативность)
  • (k₁ + k₂) · A = k₁ · A + k₂ · A (дистрибутивность)
  • k · (A + B) = k · A + k · B (дистрибутивность)

2.3 Умножение матриц

Произведение матрицы A размера m × p на матрицу B размера p × n определено только тогда, когда число столбцов матрицы A равно числу строк матрицы B. Результатом является матрица C размера m × n.

Cij = Σk=1p Aik · Bkj
где i = 1, …, m; j = 1, …, n; суммирование ведётся по k от 1 до p
Размер A: m × p, размер B: p × n → Результат: m × n

Каждый элемент результирующей матрицы Cij вычисляется как скалярное произведение i-й строки матрицы A на j-й столбец матрицы B.

Важные свойства умножения матриц:

  • Некоммутативность: A · B ≠ B · A (в общем случае)
  • Ассоциативность: (A · B) · C = A · (B · C)
  • Дистрибутивность: A · (B + C) = A · B + A · C
  • Совместимость с умножением на скаляр: k · (A · B) = (k · A) · B = A · (k · B)

2.4 Транспонирование матрицы

Транспонированная матрица AT получается из исходной матрицы A заменой строк столбцами (и наоборот). Если матрица A имеет размер m × n, то транспонированная матрица AT имеет размер n × m.

ATij = Aji
Строки исходной матрицы становятся столбцами транспонированной, и наоборот

Свойства транспонирования:

  • (AT)T = A
  • (A + B)T = AT + BT
  • (k · A)T = k · AT
  • (A · B)T = BT · AT (порядок меняется!)

2.5 Определитель матрицы

Определитель (детерминант) — числовая характеристика квадратной матрицы. Определитель обозначается det(A) или |A|. Он широко применяется для проверки обратимости матрицы, решения СЛАУ методом Крамера, вычисления объёмов и в других задачах.

Базовые случаи:

Матрица 1×1: det(A) = a11
Матрица 2×2: det(A) = a11 · a22 − a12 · a21

Разложение Лапласа по первой строке (общий случай n×n):

det(A) = Σj=1n (−1)1+j · a1j · M1j
где M1j — минор элемента a1j (определитель матрицы (n−1)×(n−1),
полученной вычёркиванием первой строки и j-го столбца)

Множитель (−1)1+j задаёт знак алгебраического дополнения: для нечётных j знак положительный (+), для чётных — отрицательный (−). Рекурсивное вычисление продолжается до тех пор, пока не будет достигнут базовый случай 1×1 или 2×2.

Свойства определителя:

  • det(A) = 0 ⇔ матрица A вырожденная (необратимая)
  • det(AT) = det(A)
  • det(A · B) = det(A) · det(B)
  • det(k · A) = kn · det(A), где n — порядок матрицы
  • При перестановке двух строк определитель меняет знак
  • Если две строки одинаковы, то det(A) = 0

2.6 Ранг матрицы

Ранг матрицы — это максимальное число линейно независимых строк (или столбцов) матрицы. Ранг обозначается rank(A) или rg(A). Ранг матрицы размера m × n не может превышать min(m, n).

Метод вычисления — элементарные преобразования строк (метод Гаусса):

  1. Приводим матрицу к ступенчатому виду (верхнетреугольному) с помощью элементарных преобразований
  2. Подсчитываем количество ненулевых строк — это и есть ранг
rank(A) = количество ненулевых строк ступенчатой матрицы
Элементарные преобразования строк:
1) Перестановка двух строк: Ri ↔ Rj
2) Умножение строки на ненулевое число: Ri → k · Ri
3) Прибавление к строке другой строки, умноженной на число: Ri → Ri + k · Rj
Теорема Кронекера-Капелли: Система линейных уравнений совместна тогда и только тогда, когда ранг матрицы коэффициентов равен рангу расширенной матрицы: rank(A) = rank(A|b).

3. Математические основы: решение СЛАУ

Система линейных алгебраических уравнений (СЛАУ) с n неизвестными и n уравнениями записывается в виде:

A · X = B
a11x1 + a12x2 + … + a1nxn = b1
a21x1 + a22x2 + … + a2nxn = b2
…………………………………………………
an1x1 + an2x2 + … + annxn = bn

3.1 Метод Гаусса

Метод Гаусса (метод последовательного исключения неизвестных) — это основной метод решения СЛАУ. Он состоит из двух этапов: прямого хода и обратного хода.

Прямой ход:

Приведение расширенной матрицы [A | b] к верхнетреугольному (ступенчатому) виду с помощью элементарных преобразований строк:

Ri → Ri − (aik / akk) · Rk
где akk — ведущий элемент (pivot) на шаге k, i > k
Цель: обнулить все элементы ниже ведущего в текущем столбце

На каждом шаге выбирается ведущий элемент (pivot). Для повышения численной устойчивости применяется частичный выбор ведущего элемента (partial pivoting) — выбирается строка с максимальным по модулю элементом в текущем столбце.

Обратный ход:

После приведения к треугольному виду значения неизвестных вычисляются последовательно, начиная с последнего уравнения:

xi = (b'i − Σj=i+1n a'ij · xj) / a'ii
для i = n, n−1, …, 1 (обратный порядок); штрих означает преобразованные коэффициенты

Алгоритм метода Гаусса:

  1. Формируем расширенную матрицу [A | b]
  2. Для каждого столбца k = 1, …, n:
  3. Находим строку с максимальным |aik| (i ≥ k) — выбор ведущего элемента
  4. Меняем строки k и найденную строку местами
  5. Для каждой строки i > k: Ri = Ri − (aik/akk) · Rk
  6. Выполняем обратный ход для нахождения xn, xn−1, …, x1

3.2 Метод Крамера

Метод Крамера применим к системам из n уравнений с n неизвестными, когда определитель матрицы коэффициентов D = det(A) ≠ 0.

xi = Di / D
где D = det(A) — определитель матрицы системы,
Di — определитель матрицы, полученной заменой i-го столбца A
на столбец свободных членов b

Для вычисления решения нужно найти n + 1 определитель: один главный определитель D и n определителей Di с заменённым столбцом.

Условия применимости метода Крамера:

  • Система должна быть квадратной (количество уравнений = количество неизвестных)
  • Определитель матрицы D ≠ 0
  • При D = 0 метод Крамера не даёт ответа (система либо несовместна, либо имеет бесконечно много решений)

3.3 Матричный метод

Матричный метод основан на нахождении обратной матрицы A−1 и умножении на неё столбца свободных членов.

X = A−1 · B
Решение существует тогда и только тогда, когда det(A) ≠ 0 (матрица обратима)

Нахождение обратной матрицы методом Гаусса-Жордана:

  1. Составляем расширенную матрицу [A | E], где E — единичная матрица
  2. Приводим левую часть к единичной матрице с помощью элементарных преобразований строк
  3. Правая часть расширенной матрицы станет обратной матрицей A−1
[A | E] → [E | A−1]
Отличие от метода Гаусса: элементы обнуляются не только ниже, но и выше ведущего элемента,
а ведущая строка делится на ведущий элемент для получения единицы на диагонали

Сравнение трёх методов решения СЛАУ:

Критерий Метод Гаусса Метод Крамера Матричный метод
Сложность O(n³) O(n! · n) O(n³)
Условие Любая квадратная система det(A) ≠ 0 det(A) ≠ 0
Наглядность Высокая Очень высокая Средняя
Побочный результат Треугольная матрица Все определители Обратная матрица A⁻¹
Устойчивость Высокая Средняя Высокая

4. Математические основы: операции над векторами

Вектор — это упорядоченный набор чисел (координат). В данном приложении вектор a⃗ = (a₁, a₂, …, aₙ) может иметь размерность от 1 до 15 для арифметических операций. Для векторного и смешанного произведений размерность фиксирована: 3 (трёхмерное пространство).

4.1 Сложение и вычитание векторов

Сложение и вычитание векторов выполняется покомпонентно. Операция определена только для векторов одинаковой размерности.

c⃗ = a⃗ ± b⃗ ⇒ ci = ai ± bi
для i = 1, 2, …, n; векторы должны иметь одинаковую размерность n

4.2 Умножение вектора на число

c⃗ = k · a⃗ ⇒ ci = k · ai
Каждая координата вектора умножается на скаляр k

Геометрический смысл: при k > 0 вектор растягивается (или сжимается), сохраняя направление; при k < 0 — направление меняется на противоположное; при k = 0 получается нулевой вектор.

4.3 Скалярное произведение

Скалярное произведение двух векторов — это число (скаляр), равное сумме произведений соответствующих координат. Используется для нахождения угла между векторами, проекций и проверки ортогональности.

(a⃗, b⃗) = Σi=1n ai · bi = a₁b₁ + a₂b₂ + … + aₙbₙ
(a⃗, b⃗) = |a⃗| · |b⃗| · cos(θ)
где θ — угол между векторами,
|a⃗| = √(a₁² + a₂² + … + aₙ²) — длина (модуль) вектора

Свойства скалярного произведения:

  • (a⃗, b⃗) = (b⃗, a⃗) — коммутативность
  • (a⃗, b⃗ + c⃗) = (a⃗, b⃗) + (a⃗, c⃗) — дистрибутивность
  • (ka⃗, b⃗) = k(a⃗, b⃗) — вынесение скаляра
  • (a⃗, b⃗) = 0 ⇔ a⃗ ⊥ b⃗ (векторы ортогональны)

4.4 Векторное произведение

Векторное произведение определено только для трёхмерных векторов. Результатом является вектор, перпендикулярный обоим исходным.

[a⃗ × b⃗] = |
  i⃗   j⃗   k⃗
  a₁  a₂  a₃
  b₁  b₂  b₃
|

Раскрывая определитель по первой строке:

[a⃗ × b⃗] = (a₂b₃ − a₃b₂) · i⃗ − (a₁b₃ − a₃b₁) · j⃗ + (a₁b₂ − a₂b₁) · k⃗
Или в координатной форме:
[a⃗ × b⃗] = (a₂b₃ − a₃b₂,  a₃b₁ − a₁b₃,  a₁b₂ − a₂b₁)

Геометрический смысл:

  • Модуль векторного произведения равен площади параллелограмма: |[a⃗ × b⃗]| = |a⃗| · |b⃗| · sin(θ)
  • Направление определяется правилом правой руки
  • [a⃗ × b⃗] = 0⃗ ⇔ a⃗ ∥ b⃗ (векторы коллинеарны)

4.5 Смешанное произведение

Смешанное произведение трёх векторов — это число (скаляр), определённое как:

(a⃗, b⃗, c⃗) = a⃗ · [b⃗ × c⃗] = det
| a₁  a₂  a₃ |
| b₁  b₂  b₃ |
| c₁  c₂  c₃ |
Равно определителю матрицы 3×3, составленной из координат трёх векторов

Геометрический смысл:

  • Модуль смешанного произведения равен объёму параллелепипеда, построенного на трёх векторах
  • (a⃗, b⃗, c⃗) = 0 ⇔ векторы компланарны (лежат в одной плоскости)
  • Знак определяет ориентацию тройки (правая или левая)

5. Реализация в коде

Все математические алгоритмы реализованы на чистом JavaScript без использования внешних библиотек. Ниже приведены ключевые функции с подробными комментариями.

5.1 Функция вычисления определителя

Определитель вычисляется рекурсивно методом разложения Лапласа по первой строке. Рекурсия имеет два базовых случая: матрица 1×1 (возвращает единственный элемент) и матрица 2×2 (формула ad − bc).

JavaScript function determinant(m, steps, depth) { depth = depth || 0; const n = m.length; const indent = '&nbsp;'.repeat(depth * 4); // Базовый случай: матрица 1×1 if (n === 1) { steps.push(`<div class="step">det = ${fmt(m[0][0])}</div>`); return m[0][0]; } // Базовый случай: матрица 2×2 if (n === 2) { const d = m[0][0] * m[1][1] - m[0][1] * m[1][0]; return d; } // Разложение по первой строке let d = 0; for (let j = 0; j < n; j++) { // Формируем минор M_{0,j} — вычёркиваем строку 0 и столбец j const minor = []; for (let i = 1; i < n; i++) { const row = []; for (let k = 0; k < n; k++) { if (k !== j) row.push(m[i][k]); } minor.push(row); } const sign = (j % 2 === 0) ? 1 : -1; // Рекурсивно вычисляем определитель минора const minorDet = determinant(minor, steps, depth + 1); d += sign * m[0][j] * minorDet; } return d; }
Принцип работы: Для матрицы n×n функция проходит по элементам первой строки. Для каждого элемента формируется минор — матрица (n−1)×(n−1) без первой строки и текущего столбца. Рекурсия продолжается, пока не достигнет матрицы 1×1 или 2×2. Алгебраические дополнения чередуют знак: +, −, +, −, …

5.2 Метод Гаусса — решение СЛАУ

Реализация включает формирование расширенной матрицы, прямой ход с частичным выбором ведущего элемента (для численной устойчивости), и обратный ход для нахождения неизвестных.

JavaScript function solveGauss(A, b, n) { // Формируем расширенную матрицу [A | b] const aug = []; for (let i = 0; i < n; i++) { aug.push([...A[i], b[i]]); } // ═══ ПРЯМОЙ ХОД ═══ for (let col = 0; col < n; col++) { // Частичный выбор ведущего элемента (partial pivoting) let maxRow = col; for (let i = col + 1; i < n; i++) { if (Math.abs(aug[i][col]) > Math.abs(aug[maxRow][col])) maxRow = i; } if (Math.abs(aug[maxRow][col]) < 1e-12) { throw new Error('Матрица вырожденная'); } // Перестановка строк if (maxRow !== col) { [aug[col], aug[maxRow]] = [aug[maxRow], aug[col]]; } // Обнуляем элементы ниже ведущего for (let i = col + 1; i < n; i++) { if (Math.abs(aug[i][col]) > 1e-12) { const factor = aug[i][col] / aug[col][col]; for (let j = col; j <= n; j++) { aug[i][j] -= factor * aug[col][j]; } } } } // ═══ ОБРАТНЫЙ ХОД ═══ const x = new Array(n).fill(0); for (let i = n - 1; i >= 0; i--) { let sum = aug[i][n]; for (let j = i + 1; j < n; j++) { sum -= aug[i][j] * x[j]; } x[i] = sum / aug[i][i]; } return x; }

Ключевые моменты реализации:

  • Расширенная матрица: массив [...A[i], b[i]] — каждая строка матрицы дополняется элементом вектора b
  • Partial pivoting: поиск максимального |aik| предотвращает деление на малые числа
  • Порог 1e-12: значения меньше 10−12 считаются нулевыми (защита от ошибок округления)
  • Деструктуризация: [aug[col], aug[maxRow]] = [aug[maxRow], aug[col]] — элегантная перестановка строк через ES6

5.3 Метод Гаусса-Жордана для обратной матрицы

Обратная матрица находится путём преобразования расширенной матрицы [A | E] в [E | A−1]. В отличие от обычного метода Гаусса, обнуление происходит как ниже, так и выше ведущего элемента.

JavaScript function inverseMatrix(m, steps) { const n = m.length; // Расширенная матрица [A | E] const aug = []; for (let i = 0; i < n; i++) { aug.push([...m[i]]); for (let j = 0; j < n; j++) { aug[i].push(i === j ? 1 : 0); // Единичная матрица } } // Для каждого столбца for (let col = 0; col < n; col++) { // Выбор ведущего элемента let maxRow = col; for (let i = col + 1; i < n; i++) { if (Math.abs(aug[i][col]) > Math.abs(aug[maxRow][col])) maxRow = i; } if (Math.abs(aug[maxRow][col]) < 1e-12) { throw new Error('Матрица вырожденная'); } if (maxRow !== col) { [aug[col], aug[maxRow]] = [aug[maxRow], aug[col]]; } // Делим строку на ведущий элемент → получаем 1 на диагонали const pivot = aug[col][col]; for (let j = 0; j < 2 * n; j++) aug[col][j] /= pivot; // Обнуляем ВЕСЬ столбец (и выше, и ниже) for (let i = 0; i < n; i++) { if (i !== col && Math.abs(aug[i][col]) > 1e-12) { const factor = aug[i][col]; for (let j = 0; j < 2 * n; j++) aug[i][j] -= factor * aug[col][j]; } } } // Извлекаем правую часть — это A⁻¹ const inv = []; for (let i = 0; i < n; i++) { inv.push(aug[i].slice(n)); } return inv; }

5.4 Метод Крамера — решение СЛАУ

Реализация метода Крамера использует функцию determinant() для вычисления главного определителя и определителей с заменённым столбцом.

JavaScript function solveCramer(A, b, n) { // 1. Главный определитель const D = determinant(A, []); if (Math.abs(D) < 1e-12) { throw new Error('D = 0, метод Крамера неприменим'); } // 2. Определители D_i const x = []; for (let j = 0; j < n; j++) { // Копируем матрицу A и заменяем j-й столбец вектором b const Aj = copyMatrix(A); for (let i = 0; i < n; i++) { Aj[i][j] = b[i]; } const Dj = determinant(Aj, []); // x_j = D_j / D x.push(Dj / D); } return x; }
Замечание: Метод Крамера использует функцию copyMatrix() для глубокого копирования матрицы, чтобы замена столбца не повлияла на исходную матрицу A. Функция copyMatrix(m) реализована как m.map(row => [...row]) — создание нового массива со spread-оператором.

5.5 Вычисление ранга матрицы

Ранг вычисляется приведением матрицы к ступенчатому виду. Количество ненулевых строк в ступенчатой матрице — это и есть ранг.

JavaScript function matrixRank(m, steps) { const M = copyMatrix(m); const rows = M.length, cols = M[0].length; let rank = 0; for (let col = 0; col < cols && rank < rows; col++) { // Ищем ненулевой элемент в столбце col let pivotRow = -1; for (let i = rank; i < rows; i++) { if (Math.abs(M[i][col]) > 1e-12) { pivotRow = i; break; } } if (pivotRow === -1) continue; // столбец нулевой // Перестановка строк if (pivotRow !== rank) { [M[rank], M[pivotRow]] = [M[pivotRow], M[rank]]; } // Обнуляем элементы ниже const pivot = M[rank][col]; for (let i = rank + 1; i < rows; i++) { if (Math.abs(M[i][col]) > 1e-12) { const factor = M[i][col] / pivot; for (let j = col; j < cols; j++) { M[i][j] -= factor * M[rank][j]; } } } rank++; } return rank; }

5.6 Векторное произведение

Векторное произведение вычисляется по формуле через миноры определителя 3×3. Каждая компонента результата — это разность произведений координат.

JavaScript case 'cross': { const b = readVector('b', 3); // Формула через компоненты определителя: // i: a₂b₃ − a₃b₂ // j: a₃b₁ − a₁b₃ (обратите внимание: знак минор j меняется!) // k: a₁b₂ − a₂b₁ const r = [ a[1]*b[2] - a[2]*b[1], // компонента i a[2]*b[0] - a[0]*b[2], // компонента j a[0]*b[1] - a[1]*b[0] // компонента k ]; // Вывод результата html += resultHTML(r); break; }
Смешанное произведение реализовано с помощью функции determinant() — из координат трёх векторов составляется матрица 3×3, и вычисляется её определитель: const m = [a, b, c]; const d = determinant(m, detSteps);

6.2 Динамическая генерация полей ввода

Одна из ключевых особенностей приложения — динамическая генерация полей ввода в зависимости от выбранной операции и размерности. При изменении типа операции или размера матрицы поля ввода полностью перестраиваются.

Как это работает:

  1. Пользователь выбирает операцию (например, «A × B»)
  2. Вызывается setMatOp('mul'), которая обновляет глобальную переменную matOp
  3. Вызывается buildMatInputs(), которая генерирует HTML для управления размерностями

  5. Функция matrixGridHTML(prefix, rows, cols) формирует CSS Grid с input-элементами

Для СЛАУ поля генерируются в виде уравнений с подписями переменных (x₁, x₂, …, xₙ), плюсами между коэффициентами и знаком «=» перед свободным членом. Это придаёт интерфейсу наглядность — пользователь видит систему уравнений в привычном виде.

Для векторов поля организованы в виде кортежа с круглыми скобками ( a₁, a₂, a₃ ) и запятыми-разделителями, что визуально соответствует математической записи вектора.

Калькулятор отображает подробные промежуточные шаги для каждого вычисления. Это ключевая учебная функция приложения, позволяющая студентам проверять своё решение на каждом этапе.

Что отображается в промежуточных шагах:

  • Сложение/вычитание: формула для каждого элемента: cij = aij ± bij = результат
  • Умножение матриц: для каждого cij показываются все члены суммы

  • Метод Крамера: вычисление каждого определителя Di
  • Матричный метод: нахождение обратной матрицы и умножение на b

Блок промежуточных шагов реализован как сворачиваемая секция. Пользователь может кликнуть на заголовок (▸ / ▾) для раскрытия или сворачивания. По умолчанию шаги развёрнуты.

7. Технические характеристики

7.1 Надёжность вычислений

Для обеспечения корректности вычислений в условиях работы с числами с плавающей запятой (IEEE 754) применяется ряд защитных механизмов:

Обработка edge-случаев:

  • Погрешность округления: значения меньше 1e-12 (10−12) считаются нулевыми, что предотвращает ложное обнаружение ненулевых элементов
  • Частичный выбор ведущего: (partial pivoting) в методах Гаусса и Гаусса-Жордана улучшает численную устойчивость
  • Нулевой определитель: проверяется перед делением в методах Крамера и матричном; при det(A) = 0 выводится информативное сообщение
  • Несовместимые размеры: интерфейс автоматически синхронизирует размеры (например, число столбцов A = число строк B при умножении)
  • Форматирование вывода: функция fmt() округляет числа до 4 знаков, убирая хвостовые нули и обрабатывая «почти целые» значения
JavaScript /** Округление числа для красивого вывода */ function fmt(x) { if (Math.abs(x) < 1e-12) return '0'; if (Number.isInteger(x)) return String(x); if (Math.abs(x - Math.round(x)) < 1e-9) return String(Math.round(x)); let s = x.toFixed(4); s = s.replace(/\.?0+$/, ''); return s; }

D = det|3, 2; 1, −1| = 3·(−1) − 2·1 = −3 − 2 = −5

D₁ = det|7, 2; −1, −1| = 7·(−1) − 2·(−1) = −7 + 2 = −5

D₂ = det|3, 7; 1, −1| = 3·(−1) − 7·1 = −3 − 7 = −10

Ответ

x₁ = D₁ / D = −5 / −5 = 1

x₂ = D₂ / D = −10 / −5 = 2

Проверка: 3·1 + 2·2 = 3 + 4 = 7 ✓    1 − 2 = −1 ✓

8.3 Примеры: операции над векторами

📌 Пример 1: Скалярное произведение

Входные данные

a⃗ = (1, 2, 3),    b⃗ = (4, −5, 6)

Вычисление

(a⃗, b⃗) = 1·4 + 2·(−5) + 3·6 = 4 − 10 + 18 = 12

Результат

(a⃗ · b⃗) = 12

📌 Пример 2: Векторное произведение

Входные данные

a⃗ = (2, 3, 4),    b⃗ = (5, 6, 7)

Вычисление по формуле

[a⃗ × b⃗] = (a₂b₃ − a₃b₂,  a₃b₁ − a₁b₃,  a₁b₂ − a₂b₁)

i: 3·7 − 4·6 = 21 − 24 = −3

j: 4·5 − 2·7 = 20 − 14 = 6

k: 2·6 − 3·5 = 12 − 15 = −3

Результат

[a⃗ × b⃗] = (−3, 6, −3)

📌 Пример 4: Смешанное произведение — компланарные векторы

  • Три модуля: матрицы, СЛАУ и векторы покрывают основные темы курса линейной алгебры

  • Обработка ошибок: информативные сообщения при некорректных данных или вырожденных матрицах
  • Чистый код: хорошо структурированный JavaScript без зависимостей от внешних библиотек
Итого: Проект демонстрирует практическое применение знаний линейной алгебры и веб-программирования. Все математические алгоритмы реализованы с нуля, с акцентом на корректность, наглядность и устойчивость вычислений.
